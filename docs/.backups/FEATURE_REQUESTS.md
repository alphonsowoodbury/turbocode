# Turbo Feature Requests

This document tracks feature requests and enhancements for the Turbo project management platform.

## Requested Features

### 1. Settings Page Improvements
**Status:** Planned
**Priority:** Medium
**Description:** The settings page needs to be fleshed out with complete functionality and options.

### 2. Issue Grouping
**Status:** Planned
**Priority:** High
**Description:** Issues need to be able to be grouped together for better organization and management.

### 3. Project Detail Page - Issue Pagination/Collapsing
**Status:** Planned
**Priority:** Medium
**Description:** The Project detail page needs to only show a limited number of issues at a time unless the user expands to see more. Improves performance and UX for projects with many issues.

### 4. Project Detail Page - Blueprint Management
**Status:** Planned
**Priority:** High
**Description:** The project detail page should display what blueprints the project is using and allow users to add/remove blueprints from the project.

### 5. Project Detail Page - Edit Functionality
**Status:** Planned
**Priority:** High
**Description:** Add ability to edit project details directly from the project detail page.

### 6. Activity Feed
**Status:** Planned
**Priority:** Medium
**Description:** Implement an activity feed showing everything that was changed in the system. Should be user configurable to filter what types of activities are shown.

### 7. Journal/Thought Entry Feature
**Status:** Planned
**Priority:** Low
**Description:** An almost journal-like thought entry feature that users can tag to projects, issues, etc. More details to be provided later.

### 8. Clickable Breadcrumbs
**Status:** Planned
**Priority:** Low
**Description:** Breadcrumbs in the navigation should be clickable links for easier navigation.

## Notes

- This list will be updated as features are implemented
- Priority levels may change based on user feedback and development progress
